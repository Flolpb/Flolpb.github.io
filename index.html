Tests unitaires

Les tests unitaires présent sur l'application permettent de tester les fonctions de l'application pour être sur qu'il n'y a pas de bug avant de finalement la déployer sur un environnement de production.

Tests de sécurité

Les tests de sécurité fournit par Trivy nous permettent de scanner les différents packages npm installés sur l'application express et trouver les failles de sécurité les concernant. Ils peuvent aussi scanner des conteneurs ou encore des scans de dépendance.

Tests de qualité

Les tests de qualité sont utiles pour détecter les bugs, les duplications de code, les codes non optimisés, la couverture de tests et les vulnérabilités. En résumé, les tests de qualités sont la pour valider la qualité globale du code.

Difficultées rencontrées

Nous n'avons rencontré aucune réelle difficulté. Le plus compliqué pour nous était de savoir comment nous allions déployer sur deux environnements différents et où est-ce qu'on allait les déployer.

Déploiement

On a réussi à déployer notre site en passant par Firebase car ils offrent le choix d'héberger plusieurs sites webs sur un seul projet Firebase tout en restant gratuit dans leur offre. On a donc opter pour un déploiement dans le cloud.

Paradigmes

Un des paradigmes exploités lors de ce tp est celui de la collaboration entre la partie développement et la partie obs. Le deuxième paradigme majeur est celui de l'automatisation par le biais du CI/CD et un des autres paradigmes c'est celui du testing et du reviewing de code qui permet de conserver une qualité de service et de détecter les failles eventuelles du projet.

Etat de l'art

Une pipeline DEVOPS permet d'intégrer le code pour ensuite lancer une phase de test pour ce dernier et ensuite le délivrer (déployer + publier). On peut avoir des pipelines seulement pour le test, et d'autres qui délivrent donc déploient et publient l'application automatiquement.
